1. El Box Model

Todo elemento HTML es una caja con 4 capas: contenido, padding (relleno interno), border (borde) y margin (espacio externo). Los colores son los mismos que usa Chrome DevTools cuando inspeccionás un elemento.

3. Flexbox: los hijos (flex ítems)

    

      Las propiedades de la sección anterior se escriben en el contenedor.
      Pero cada hijo (cada flex ítem) tiene las suyas:
      order, flex-grow, flex-shrink y
      flex-basis. Tocá una caja para seleccionarla
      y modificala con los controles — cada ítem guarda sus propios valores.

      💡 Ítem y contenedor a la vez: un elemento puede ser flex ítem
      (hijo de un contenedor flex) y además flex container (tener display: flex
      para ordenar a sus propios hijos). Es el único caso en el que un elemento
      lleva propiedades de los dos mundos. De hecho, las cajas de arriba son exactamente
      eso: cada .item es hijo del contenedor punteado y usa
      display:flex para centrar su número y su etiquetita.
    

  



  

    
4. Pseudoclases: estilos según el estado

    

      Una pseudoclase se escribe con dos puntos (selector:estado) y aplica
      estilos solo cuando el elemento está en ese estado. La más famosa
      es :hover. Armá tu propio efecto con transform +
      box-shadow y probalo pasando el mouse por la tarjeta.

5. Medidas: absolutas vs. relativas

    

      Una medida absoluta (px) mide siempre lo mismo, se vea donde se vea.
      Una relativa (%, rem, em, vw, vh) depende de otra cosa — del padre,
      de la raíz o de la ventana — y por eso se adapta sola.
      Achicá el contenedor y fijate cuál de las dos barras sobrevive.

width: 300px → absoluta, no se entera de nada

            
width: 70% 

          

        


        
        

          
Este texto mide 16px fijos: ignora al usuario.

          
Este mide 1em: acompaña el tamaño elegido 

        


        
/* ABSOLUTA: mide siempre lo mismo */
.foto { width: 300px; }       /* en un celular... se desborda 💥 */

/* RELATIVAS: dependen de otra cosa, por eso se adaptan */
.tarjeta { width: 70%; }      /* % → del ancho del PADRE */
p     { font-size: 1.2rem; }  /* rem → del font-size de <html> (16px por defecto → 19.2px) */
small { font-size: 0.8em; }   /* em → del font-size del PADRE directo */
.portada { height: 100vh; }   /* vh/vw → de la VENTANA (viewport) */

      

    


    

      💡 ¿Por qué preferimos relativas? Porque tu página se va a ver en
      mil pantallas distintas, y porque hay gente que agranda la letra por accesibilidad:
      rem y em la respetan, px no. Guía rápida:
      % mira al padre, em mira al font-size del padre,
      rem mira siempre a la raíz (<html>), y
      vw/vh miran a la ventana. Los px quedan para
      detalles que no deben cambiar: bordes finos, sombras.

🧹 Antes de empezar: el reset

    

      Los navegadores traen estilos propios "de fábrica": el <body> tiene
      margen, los títulos tienen tamaños predefinidos, las listas traen sangría… y cada
      navegador lo hace un poquito distinto. El reset borra todo eso para
      arrancar de cero y que tu sitio se vea igual en todos lados. Van al
      principio de tu CSS estas tres líneas:
    

    
/* RESET: va primero en el archivo CSS */
* {                            /* el asterisco selecciona TODOS los elementos */
  margin: 0;                 /* borra los márgenes de fábrica */
  padding: 0;                /* borra los rellenos de fábrica */
  box-sizing: border-box;  /* el width incluye padding y borde (sección 1) */
}


    
    
📐 Espaciado: lado por lado

    

      margin y padding se pueden aplicar a un solo lado
      agregando -top, -right, -bottom o -left:
    

    

      
margin-topMargen solo arribamargin-top: 20px;

      
margin-bottomMargen solo abajo (el más usado: separa un elemento del siguiente)margin-bottom: 1rem;

      
padding-leftRelleno solo a la izquierda (y así con right, top, bottom)padding-left: 12px;

      
margin: 0 autoTruco clásico: centra horizontalmente un bloque que tiene widthmargin: 0 auto;

    

    

      Y el atajo con varios valores sigue el sentido del reloj 🕐,
      empezando desde arriba:
    

    
/* 4 valores: arriba → derecha → abajo → izquierda (como el reloj) */
margin: 10px 20px 30px 40px;

/* 2 valores: vertical (arriba y abajo) | horizontal (izquierda y derecha) */
padding: 10px 25px;

/* 1 valor: los cuatro lados iguales */
margin: 16px;

Bloque con width: 60% + margin: 0 auto: el margen sobrante se reparte igual a los costados.

font-sizeTamaño de letra (mejor en rem, sección 5)font-size: 1.2rem;

🎨 Cómo se escriben los colores

    

      
por nombre Hay ~140 nombres en inglés. Fáciles para probar, limitados para diseñarcolor: tomato;

      
hexadecimal El formato más usado: # + 6 caracteres (rojo-verde-azul). Es el que copiás de cualquier selector de colorcolor: #2563eb;

      
rgb() Lo mismo pero con números de 0 a 255rgb(37, 99, 235)

      
rgba() Igual que rgb + un cuarto valor de transparencia (0 = invisible, 1 = sólido). Ideal para sombras y fondos suavesrgba(37, 99, 235, 0.4)

    


    
    
🖼️ Fondos, bordes y sombras

    

      
background-colorColor de fondo del elementobackground-color: #f6f7f9;

      
borderAtajo de tres partes: grosor + estilo (solid, dashed, dotted) + colorborder: 2px solid #1c2530;

box-shadowSombra: desplazamiento-x, desplazamiento-y, difuminado, color (con rgba para que sea suave)box-shadow: 0 4px 12px rgba(0,0,0,.15);

    


    
    
🔘 Listas (y el truco del menú)

    

      list-style: none le saca las viñetas a una lista. Es el primer paso del
      patrón más repetido de la web: todo menú de navegación es un
      <ul> sin viñetas puesto en fila con flexbox.
    

    
/* De lista con viñetas → a menú horizontal */
nav ul {
  list-style: none;   /* 1. chau viñetas */
  display: flex;      /* 2. los <li> se ponen en fila (¡sección 2!) */
  gap: 1.5rem;        /* 3. separación entre opciones */
}
nav a {
  text-decoration: none;  /* 4. chau subrayado */
}
